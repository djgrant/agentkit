## Markdown

Do not "format" markdown files with mid-sentence line breaks.

## Code changes

When asked to make a change, never make unilateral decisions to provide fallbacks. If you are asked to change something, burn the old code and docs with fire.

Do not mention "codex" in branch names, commits or pull requests.

## Dialogue

If the user request is not clear, ask for more information.

If the user request is misinformed, let the user know directly.
